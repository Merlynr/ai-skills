# 高软学习会话

> 通用框架见 `unified-learning-session`；本 Skill 定义高软专用路径、题面格式与 nmem 简报。

## 用户要求（须遵守）

1. **开启会话**：创建/打开当日学习记录，说明发题方式，链到日计划与阶段计划。
2. **每道错题**：记录**全部内容**（题干、选项、你的答案、正确答案、解析）；错因写入「错因归类」表。
3. **截图必存**：题目图片复制到 `assets/YYYY-MM-DD/`，学习记录嵌入 `![[assets/...]]`。
4. **知识点双链**：错题/对题均归档到 `01 知识点/`；**相关题目标题必须链回学习记录题目标题**；学习记录「知识点索引」含考点 + 「本题详情」双链（见 §B.1）。
5. **全量归纳**：题干 + **全部选项 A/B/C/D** 均写入；**对题 / 错题 / 提问** 覆盖相同、**侧重不同**（见 §B.2）。
6. **联网补充**：概念/标准/对比不清时先 WebSearch，结合答案写入「基本概念」（见 §B.2）。
7. **会话结束**：同步计划 + 写入 nmem 简报（UTF-8，MCP 优先）。

日期默认 **Asia/Shanghai** 当天。

## 工作流

### A. 开启会话

1. 读 `系统架构师二战学习计划.md` 的 **当前进度指针**。
2. 读 `10 Daily/YYYY-MM-DD.md` 中高软项。
3. 若不存在则创建 `03 错题/YYYY-MM-DD 高软学习记录.md`（结构见 [template.md](template.md)）。
4. 在日计划 `## Notes` 增加学习记录链接。
5. 告知用户：可发文字或截图；建议含题号、选项、你的答案、标准答案。

### B. 处理每道题（错题 / 巩固对题）

```
- [ ] 复制截图 → assets/YYYY-MM-DD/第N题-简述.png
- [ ] 学习记录追加条目（## 第N题（章节 · 考点）+ ![[assets/...]]）
- [ ] 更新「刷题记录」表
- [ ] 错题：更新「错因归类」表
- [ ] **全量提取**题干 + A/B/C/D 全部选项（§B.2）；必要时 **WebSearch** 补概念
- [ ] 追加/更新 01 知识点/（含「本题知识覆盖」表 + 选项逐项归纳 + 双链，§B.1）
- [ ] 更新学习记录「知识点索引」（考点 + 本题详情）
- [ ] nmem：memory_add 错题/对题记录（MCP，勿管道 stdin）
```

**单题条目结构**（参照 `2026-05-21 高软学习记录`）：

- `## 第N题（章节 · 考点）` + 配图（**此标题即 Obsidian 锚点**）
- 题目 / **选项表格（A/B/C/D 每项：内容 + 判据 + 理由）** / 答案
- **你错在哪**（错题必写：误选为何看似合理、与正选差在哪一步）
- 解题思路（分步；**提问时先直接答用户疑问**）
- 相关内容（表格、对比、公式；可含联网补充要点）
- 记忆点（覆盖全部选项；**错题额外写「勿再选 X 因为…」**）

**巩固题**：写入「刷题巩固」，标题 `### 第N题 · 简述`；**同样全选项归纳**，仍要配图、双链与 01 知识点归档。

#### B.1 Obsidian 双链规范（必守）

| 方向 | 写法 |
| --- | --- |
| **知识点 → 题目** | `#### [[20 Projects/高软-系统架构师/03 错题/YYYY-MM-DD 高软学习记录#第N题（章节 · 考点）\|第N题]]（YYYY-MM-DD，错题/对题）` |
| **学习记录 → 知识点** | `- 第N题 → [[01 知识点/第X章-章节名#考点名\|考点名]] · [[20 Projects/高软-系统架构师/03 错题/YYYY-MM-DD 高软学习记录#第N题（章节 · 考点）\|本题详情]]` |

- 锚点 `#` 与学习记录中题目标题**完全一致**
- 禁止知识点里只写「题目1（日期）」无链接

#### B.2 全量归纳 + 侧重策略 + 联网补充（必守）

**原则**：三种输入 **覆盖范围相同**（题干 + A/B/C/D 全写），**叙述篇幅按类型倾斜**。

| 类型 | 01 知识点 + 学习记录侧重 |
| --- | --- |
| **对题** | 正确项为何对；各干扰项为何错；均衡对比表 |
| **错题** | **你的误选**（✗ 标出）→ 为何错；**正确答案** → 为何对；**误选 vs 正选**一步之差；错因 + 回炉动作 |
| **提问** | **先答用户疑问**（一句话结论）；再关联到具体选项/概念；其余选项简写但不省略 |

**禁止**：
- 对题只写 C 而对
- 错题只写「你选 C 错了，应该 B」而跳过 A/D
- 提问只答一句不归档到 01 知识点

**必做清单**（四项选项均必填表）：

| 对象 | 要求 |
| --- | --- |
| 题干 | 考点主题、问法（正确/错误/不包括） |
| A / B / C / D | 概念 + 本题对/错 + 理由（错题在误选项加粗） |
| 本题侧重 | 见上表，写入 01 知识点 `### 本题侧重` |
| 易混淆对比 | A/B/C/D 差异；错题重点写 **误选↔正选** 对 |

**联网查询**（下列情况必须先查再写）：

- 协议/架构/存储/范式等术语需核实定义
- 多选项构成易混对（DAS/NAS/SAN、FC SAN/IP SAN、同步/互斥…）
- 干扰项或**误选涉及**的概念陌生
- 用户提问点超出题库解析深度

检索词：`软考 系统架构师 [考点]`、`[概念A] [概念B] 区别`。

**知识点文件片段 — 对题示例**：

```markdown
## 考点: 网络存储 DAS·NAS·SAN

### 基本概念
- **DAS**：SCSI 直连，无独立 OS，不跨平台共享
- **NAS**：网络接口，自带 OS，文件级
- **SAN**：块级；**FC SAN**=光纤通道，**IP SAN**=以太网（iSCSI）

### 本题知识覆盖
| 来源 | 知识点 | 本题判据 |
| 题干 | 网络存储叙述 | 选正确项 |
| A | DAS 跨平台共享 | ✗ DAS 不能跨平台 |
| B | NAS 经 SCSI 连服务器 | ✗ NAS 走网络，SCSI 是 DAS |
| C | FC SAN / IP SAN 介质 | ✓ |
| D | SAN 有 FS、NAS 无 FS | ✗ NAS 自带 OS |

### 相关题目
#### [[20 Projects/高软-系统架构师/03 错题/2026-06-04 高软学习记录#第18题 · 网络存储（DAS / NAS / SAN）|第18题]]（2026-06-04，对题）
```

```

```

1. **学习记录**：`status: completed`；今日目标 `[x]`；进度表「已完成」；今日小结。

### D. 写入 Nowledge Mem 简报（必做，防乱码）

#### 写入步骤

1. **生成 UTF-8 文件**（无 BOM）：`.cursor/skills/gaoruan-study-session/nmem-briefing-YYYY-MM-DD.md`
2. **MCP `memory_add`**（首选）：
   - `id`：`gaoruan-study-YYYY-MM-DD`
   - `title`：`高软学习简报 YYYY-MM-DD （章节·第N批）`
   - `labels`：`gaoruan,study,soft-exam`
   - `importance`：`0.7`；`event_start`：`YYYY-MM-DD`
3. **禁止**：`$text | nmem m add --stdin`（Windows 易乱码）
4. **验证**：`nmem m show gaoruan-study-YYYY-MM-DD`

## 节奏与计划原则

- 默认 **15 题/工作日、约 45 分钟**；按 **批次** 勾选。
- 小节多批时只勾当前批，整节全部批完成后再勾父级。

## 领域速查

| 考点 | 要点 |
| --- | --- |
| 计数信号量 | 初值 = 资源数 N；S<0 时等待数=\|S\| |
| 段式地址 | 合法当 0 ≤ 段内地址 < 段长 |
| 页式地址 | 只换页框号，**页内偏移照抄** |
| 层次化网络 | **接入先行**；禁额外连接；FC/IP SAN |
| 网络存储 | DAS 不共享；NAS 有 OS；SAN 块级分 FC/IP |
| 实时/嵌入式 | 外部事件须在**被控对象允许时间内** |
