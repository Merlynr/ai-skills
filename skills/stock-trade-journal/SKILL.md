# 股票每日实操反思

## 目标

1. 生成或更新：`20 Projects/天才交易员/04-实操日志/YYYY-MM-DD 实操反思.md`
2. **UZI 证据优先**：涉及股票从 [uzi-reports.md](uzi-reports.md) 读取；读毕 **归档到笔记库** [uzi-archive.md](uzi-archive.md)
3. 笔记写完后输出：**后续操作建议** + **能力缺陷** + **加强学习清单**（对齐课程与任务板）

登记标准：`04-实操日志/README.md` · 模板：`_模板-每日实操与反思.md` · 路由：[course-routing.md](course-routing.md) · UZI 读取：[uzi-reports.md](uzi-reports.md) · UZI 归档：[uzi-archive.md](uzi-archive.md)

笔记库根：当前 workspace 为 `f:/note` 时用相对路径；否则用 `f:/note/20 Projects/天才交易员/...`。

---

## 触发后固定顺序（不可跳过）

### 第 1 步：读证据（静默 · 禁止先写日志文件）

**硬性要求**：必须用 **Read** / **Grep** / **Glob** 读取原文。禁止凭记忆复述铁律、上篇内容或 UZI 数据。读失败则中止并报告路径。

| 序号 | 动作 | 说明 |
|:----:|------|------|
| 1.1 | 定日期 `D` | 用户指定 > 否则今天（Asia/Shanghai） |
| 1.2 | 找**上一篇有效日志** | 见「上篇查找算法」 |
| 1.3 | Read 铁律 SSOT | `01-基础知识-学习笔记/07-非对称风险-笔记.md`（铁律 1–6） |
| 1.4 | Read 上一篇全文 | 上一篇 `*实操反思.md`（若有） |
| 1.5 | 抽「昨日对今日的期望」 | ⑦明日一条 > ①假设/三锚 > frontmatter/④ |
| 1.6 | 读上篇 `violated_rules` | 供连续违规警告 |
| 1.7 | Read `00-学习清单.md` | 当前 S 阶段 |
| 1.8 | Read `股票投资系统学习任务.md` | **仅**「进度仪表盘 + 当前阶段章节 + 第一个未勾选项」 |
| 1.9 | 无上篇日志 | 用学习清单/日计划中的「只观察」等为期望参照 |
| **1.10** | **收集今日标的列表** | 用户口述 + 截图 + 上篇持仓 `symbols`；含已清仓/仅观察 |
| **1.11** | **读 UZI 证据（每只）** | 按 [uzi-reports.md](uzi-reports.md) **笔记库优先** → 缺则 APPDATA reports/cache |
| **1.11a** | **归档 UZI 到笔记库（每只·强制）** | 按 [uzi-archive.md](uzi-archive.md) 跑 `scripts/archive_uzi_snapshot.py {TICKER}`；失败则手工复制 one-liner + 写 `snapshot.json` |
| **1.12** | **（可选）** Read `00-当前持仓对照.md` | 对照既有铁律执行单与 `uzi-snapshots/` 链接 |

#### 上篇查找算法

**禁止**日历 `D-1`。在 `04-实操日志/` 取 `*实操反思.md`，解析 `date`，**`d < D` 倒序第一篇**。

### 第 2 步：输出「评价基准」（写文件之前）

在原有基准块之外，**必须增加**：

```markdown
### UZI 盘面快照（{报告日期}）
| 代码 | 现价 | 评分 | 定调 | 模型止损 | 关键矛盾（one-liner/punchline 缩略） |
|------|-----:|:----:|------|----------|-------------------------------------|
| ... | 来自报告，禁止编造 | | | | |
```

然后输出：连续违规警告 + 铁律 1–6 + 上篇期望 + 综合判定顺序表 + 登记标尺（见 [iron-rules.md](iron-rules.md)）。

#### 分支 · STOP

用户**未**给今日操作明细 → 基准块（含已有 UZI 快照）+ 提问 + **`[WAITING_FOR_USER]`** → **STOP**（禁止写日志、禁止第 3–5 步）。

### 第 3 步：评价并写入日志

（双线登记、`signal_source`、`manual_override` 反思小节、`overall_grade` 严格顺序 — 见 [iron-rules.md](iron-rules.md)）

#### 强制章节 · UZI × 用户描述

**§四 效果评价** 下必须有：

```markdown
### 盘面核对 · 你的理解对不对（UZI + 口述）

| 标的 | 你的理解 / 操作 | UZI/盘面验证 | 结论 |
|------|-----------------|--------------|:----:|
| {代码} | 用户原话、买卖价 | 报告现价、日K关键位、LHB、DCF矛盾、成交价对比 | ✅/❌/🟡 |
```

规则：

- 每一只当日涉及的标的**至少一行**；数据来自第 1.11 步（笔记库 `snapshot.json` 或 live 报告），**禁止**未读就填「UZI 显示」  
- 用户截图中的成本/盈亏与报告现价可同时引用  
- 「朋友推荐/价投叙事」标为用户信源，用 UZI 基本面/评委/杀猪盘反驳或印证  

**§八 关联** 须链 **笔记库归档** `[[uzi-snapshots/{TICKER}/{date}/snapshot]]`（主）；APPDATA HTML 可作辅链并标注「可能过期」。另链 `[[00-当前持仓对照]]` / `[[00-UZI*股分析*]]`。

日志 frontmatter 增加（有归档时）：

```yaml
uzi_snapshots:
  - "600121.SH/20260608"
```

填写模板 **「九、课程对齐 · 后续建议」** 骨架（可先占位，第 5 步补全）。

**文件已存在**：只更新相关节；**不存在**：Read 模板后创建。

### 第 4 步：登记收尾（简短）

- 文件路径 · 效果/执行/综合一行 · 相对上篇预期 · 连续违规编号  
- **一行 UZI**：已对照 {N} 只 · 报告截至 {最新日期} · **已归档** {N} 只至 `uzi-snapshots/` · 缺报告列表（若有）

### 第 5 步：课程对齐复盘（写完笔记后 · 必读课程证据）

**触发**：第 3 步日志已写入（或已更新）后**必须执行**。向用户输出独立块 **`## 📚 课程对齐 · 后续建议（D）`**，并**回写**日志「九、课程对齐」三节。

#### 5.1 必读证据（至少 3 个文件 + UZI）

1. 今日 `violated_rules`、反思关键词 → [course-routing.md](course-routing.md) 选 **Read 笔记章节**  
2. **持仓逐只**：复述第 1.11 步 UZI 现价/止损/目标 + [uzi-reports.md §7](uzi-reports.md) 格式写明日触发  
3. 铁律止损 **优先于** UZI 模型 Stop（须在建议中体现）

| 主题 | 常见路径 |
|------|----------|
| 止盈/量价 | `10-量能入门-笔记.md`、`03-看盘界面-笔记.md` |
| 风控/仓位 | `07-非对称风险-笔记.md`、`02-资金流动-笔记.md` |
| 板/溢价 | `09-手机盯盘-笔记.md`、`02-案例研究/17交易模式拆解-笔记.md` |
| K 线退出 | `K线基础知识-笔记.md` |
| 筹码/成本 | `筹码理论-笔记.md` |
| 任务顺序 | `股票投资系统学习任务.md`、`00-学习清单.md` |

**禁止**：未 Read 就写课程建议；未读 UZI 就写价位触发；泛泛「注意风险」。

#### 5.2 输出结构（强制四节）

```markdown
## 📚 课程对齐 · 后续建议（YYYY-MM-DD）

### 一、后续操作建议（下一交易日）
- 按**当前持仓**逐只：**UZI 快照一行** + **铁律触发价 → 动作（减%/清仓/观望）**
- 账户级：仓位上限、暂停新开仓、模拟 vs 实盘（铁律 6 + S 阶段）
- 至少 1 条延续今日「明日一条」

### 二、暴露的能力缺陷（今日证据）
- 3–5 条；格式：**缺陷** — 现象（日志/铁律/UZI 对照）

### 三、建议加强的学习
- 每条：**学什么** · **为何** · **去哪学**（`[[笔记#章节]]` 或 `S?-T??`）· **建议用时**

### 四、任务板指针
- 当前阶段 / 指针 / 实盘与 S0–S4 冲突 · 明天任务建议
```

#### 5.3 写回笔记（除日志外）

- 将 5.2 写入当日 `## 九、课程对齐 · 后续建议`  
- **同步** `04-实操日志/00-当前持仓对照.md` §五「明日一条」与 §三各股执行单（持仓有变时）  
- **确认** 第 1.11a 步归档完成；更新 `uzi-snapshots/README.md` 索引  
- 若 ≥3 只且需留档：更新 `00-UZI{N}股分析-{日期}.md`，底部链 `uzi-snapshots/{TICKER}/{date}/`  

#### 5.4 原则

- **UZI + 铁律**：操作建议 = 铁律触发为主，UZI 定调/模型价为辅  
- **认知≠执行**：看懂 UZI/盘面但没做到 → 缺陷归执行，补课偏 10 课、S2  
- **阶段门控**：S1 + `live` → 减仓 + 模拟；DCF「低估」不得作为亏损死扛理由  
- **过程优先**：不得鼓励违反今日已判 ❌ 的铁律
