# 股票每日实操反思

## 目标

1. 生成或更新：`20 Projects/天才交易员/04-实操日志/YYYY-MM-DD 实操反思.md`
2. 笔记写完后输出：**后续操作建议** + **能力缺陷** + **加强学习清单**（对齐课程与任务板）

登记标准：`04-实操日志/README.md` · 模板：`_模板-每日实操与反思.md` · 路由：[course-routing.md](course-routing.md)

笔记库根：当前 workspace 为 `f:/note` 时用相对路径；否则用 `f:/note/20 Projects/天才交易员/...`。

---

## 触发后固定顺序（不可跳过）

### 第 1 步：读证据（静默 · 禁止先写日志文件）

**硬性要求**：必须用 **Read** 读取原文。禁止凭记忆复述铁律或上篇内容。读失败则中止并报告路径。

| 序号 | 动作 | 说明 |
|:----:|------|------|
| 1.1 | 定日期 `D` | 用户指定 > 否则今天（Asia/Shanghai） |
| 1.2 | 找**上一篇有效日志** | 见「上篇查找算法」 |
| 1.3 | Read 铁律 SSOT | `01-基础知识-学习笔记/07-非对称风险-笔记.md`（铁律 1–6） |
| 1.4 | Read 上一篇全文 | 上一篇 `*实操反思.md`（若有） |
| 1.5 | 抽「昨日对今日的期望」 | ⑦明日一条 > ①假设/三锚 > frontmatter/④ |
| 1.6 | 读上篇 `violated_rules` | 供连续违规警告 |
| 1.7 | Read `00-学习清单.md` | 当前 S 阶段 |
| 1.8 | Read `股票投资系统学习任务.md` | **仅**「进度仪表盘 + 当前阶段章节 + 第一个未勾选项」 |
| 1.9 | 无上篇日志 | 用学习清单/日计划中的「只观察」等为期望参照 |

#### 上篇查找算法

**禁止**日历 `D-1`。在 `04-实操日志/` 取 `*实操反思.md`，解析 `date`，**`d < D` 倒序第一篇**。

### 第 5 步：课程对齐复盘（写完笔记后 · 必读课程证据）

**触发**：第 3 步日志已写入（或已更新）后**必须执行**。向用户输出独立块 **`## 📚 课程对齐 · 后续建议（D）`**，并**回写**日志「九、课程对齐」三节（与聊天内容一致）。

#### 5.1 必读证据（按缺陷选读，至少 3 个文件）

用今日 `violated_rules`、反思关键词、持仓问题，从 [course-routing.md](course-routing.md) 选路由，**Read 对应笔记章节**（带 `#` 锚点或整节），例如：

| 主题 | 常见路径 |
|------|----------|
| 止盈/量价 | `10-量能入门-笔记.md`、`03-看盘界面-笔记.md` |
| 风控/仓位 | `07-非对称风险-笔记.md`、`02-资金流动-笔记.md` |
| 板/溢价 | `09-手机盯盘-笔记.md`、`02-案例研究/17交易模式拆解-笔记.md` |
| K 线退出 | `K线基础知识-笔记.md` |
| 筹码/成本 | `筹码理论-笔记.md` |
| 任务顺序 | `股票投资系统学习任务.md`、`00-学习清单.md` |

**禁止**：未 Read 就写「建议学第 X 课」；禁止泛泛「加强心态」而无笔记/任务依据。

#### 5.2 输出结构（强制四节）

```markdown
## 📚 课程对齐 · 后续建议（YYYY-MM-DD）

### 一、后续操作建议（下一交易日）
- 按**当前持仓**逐只：触发条件（价格/量能/时间）+ 具体动作（买/卖/减%/观望）
- 账户级：仓位上限、是否暂停新开仓、模拟 vs 实盘（对照铁律 6 与 S 阶段）
- 至少 1 条来自今日「明日一条」的延伸

### 二、暴露的能力缺陷（今日证据）
- 3–5 条；格式：**缺陷** — 今日现象（引用日志/铁律编号）

### 三、建议加强的学习
- 每条：**学什么** · **为何**（对应上节缺陷）· **去哪学**（`[[笔记#章节]]` 或 `S?-T??`）· **建议用时**
- 至少 1 项来自任务板「当前第一个未完成」；至少 1 项来自笔记补课

### 四、任务板指针
- 当前阶段 / 指针 / 今日实盘是否与「S0–S4 不下单」冲突 · 建议明天是否做 S1-T03 等
```

#### 5.3 写回日志

将 5.2 全文（可略删聊天口语）写入当日文件的 `## 九、课程对齐 · 后续建议` 三个子节。

#### 5.4 原则

- **过程优先**：操作建议不得鼓励违反今日已判 ❌ 的铁律（如建议满仓赌反弹）。
- **认知≠执行**：若用户「看懂但没做到」，缺陷归 **执行/止盈规则**，补课偏 10 课、S2 实操，而非再堆理论。
- **阶段门控**：S1 且今日 `live` → 建议 **减仓 + 模拟复盘同一盘面**，并指向 S1-M01 前任务。
